## C. Issues P0 (Ação cartorio-n8n nas próximas sprints)

### C.1 — `$env.X` broken in 7+ cron workflows

**Root cause**: N8N community license não inclui `feat:variables`. Ao tentar `$env.EVOLUTION_API_KEY` em HTTP Request node, N8N retorna "access to env vars denied".

**Workflows afetados**: WF #21, #22, #23 (cron stale-detector), #25 (metrics + protocolo-concluido), #29, #30 (health-deep-check).

**Workaround atual**: Workflows usam header com valor **hardcoded** (`X-API-Key: cartorio-api-shared-secret-v1`) em vez de `$env.CARTORIO_API_KEY` — funciona mas é débito técnico (rot 90d quebra silenciosamente).

**Soluções candidatas**:
- (A) Cartorio-n8n cria N8N Variables workaround via `.env` injected no container Docker (já feito parcialmente; checar se faltam envs)
- (B) Substituir $env por literal via `N8N_API_KEY` + atualizar workflow via API (mais simples mas perde rotação)
- (C) Upgrade para N8N enterprise ($$$, bloqueador)

**Recomendação**: Gustavo GO em (A) — auditar `docker service inspect cartorio_n8n` para confirmar EVOLUTION_API_URL, EVOLUTION_API_KEY, CARTORIO_API_KEY presentes; se faltam, fazer `docker service update --env-add` para cada. Pós-fix, reativar workflows com cron.
