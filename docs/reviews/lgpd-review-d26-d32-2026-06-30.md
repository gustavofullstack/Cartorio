| **Proteção delete duplo** | ⚠️ **GAP** | Se `direito_esquecimento()` retornar `"erro": "cliente_nao_encontrado"`, o endpoint retorna 404. Mas se o cliente JÁ foi deletado, o 404 mascara o fato. O correto seria distinguir "não existe" vs "já deletado". |

### Gap D28-G1 (ALTA)
```python
# lgpd_direitos_v2.py:455
if result.get("erro") == "cliente_nao_encontrado":
    raise HTTPException(status_code=404, ...)
```
**Problema**: Se o cliente foi deletado anteriormente, este endpoint retorna 404 indistinguível de "cliente nunca existiu".
**Recomendação**: Adicionar campo `result.get("erro") == "cliente_ja_deletado"` → retornar 410 Gone com mensagem clara.
**Estimativa**: 3 linhas no service + 1 novo erro no endpoint. 15min.

## D29 — Portabilidade/Export (art. 18 V) ⚠️ **GAP CRÍTICO P0**

**Conceito**: Exportar todos os dados do titular.

### ⚠️ D29-G1 (CRÍTICA — P0)

**Problema**: `exportar_dados_titular()` retorna `bundle.cliente` com dados PII **sem mascaramento** no response HTTP.

```python
# lgpd_direitos_v2.py:544-558
return {
    "status": "ok",
    "dados": {
        "cliente": bundle.cliente,  # ← PII SEM MÁSCARA
        "protocolos": bundle.protocolos,
        ...
    },
}
```

**Impacto LGPD**: Qualquer pessoa com JWT do titular (mesmo scopes limitados) consegue expor nome, email, telefone, CPF hash via `/lgpd/export/{cliente_id}`. Mesmo que CPF seja hash, o bundle pode conter `cpf_hash` que com rainbow table vira reversível.
**Correção obrigatória**:
1. Máscarar campos PII no `bundle.cliente` antes de retornar (CPF hash: mostrar `***.XXX.XXX-YY`; email: `j***@gmail.com`; telefone: `(11) *****-1234`).
2. Adicionar header `X-Content-Type-Options: nosniff` e `Cache-Control: no-store` no endpoint.
3. Log de acesso ao export deve conter IP completo (já tem via `audit_kwargs`).
**Estimativa**: 20min (mudar `lgpd_export.py` para retornar PII mascarado no bundle, não só na renderização).

### ⚠️ D29-G2 (ALTA)

**Problema**: `exportar_dados_titular()` pode não estar aplicando máscara nos campos `bundle.cliente`. O mesmo vale para `download_portabilidade` (v1).

O download v1 (lgpd_direitos.py:290-348) também retorna `bundle.cliente` sem máscara:

```python
# lgpd_direitos.py:334-347
return {
    "dados": {
        "cliente": bundle.cliente,  # ← PII SEM MÁSCARA
        ...
    },
}
```

**Recomendação**: Padronizar que `lgpd_export.py` NUNCA retorna PII bruto — retorna sempre máscara na struct do bundle. Quem chama (D29, download v1) recebe máscara automaticamente.
