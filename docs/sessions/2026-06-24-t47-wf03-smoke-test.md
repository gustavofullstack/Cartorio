## Licao adicional

Restart do N8N-runner sozinho nao resolve bug systemic de idle-timeout. Para projetos N8N 2.x com cron workflows:
- SEMPRE configurar `N8N_RUNNERS_LAUNCHER_AUTO_SHUTDOWN=false` (ou timeout grande)
- OU usar N8N_RUNNERS_MODE=internal (launchers embedded no main, sem auto-shutdown)
- Trade-off: launcher embedded consome mais CPU/RAM no main container
