## DNS antes de auth

- Antes de assumir problema de auth (user/password/session), sempre verificar DNS primeiro (HTTP 000, NXDOMAIN, connection refused = rede, nao auth). Hipotese mais simples primeiro: usuario esqueceu senha > bug de config.

## Credenciais em chat = queimadas

- ZERO plaintext/hashes em canal logado mesmo em incident response. SSH + env var inject ou 1Password/Bitwarden/Telegram-auto-delete. Cred queimada (mesmo prefixo/nome) = rotacionar.
  source: memory/MEMORY.md:18-33

## N8N local files != prod

- Arquivos em infra/n8n-workflows/ sao staging/templates. Antes de editar, validar WF ID de prod via `n8n export:workflow --id=X`. Editar staging achando que eh prod = nunca chega em prod.
  source: memory/MEMORY.md:35-42

## Pai pode ter contexto git stale

- 3 checks antes de aceitar commit em massa: `git status -uall`, `git ls-files`, `git log master -5`. Se clean+tracked, reportar BLOCK com evidencia, nao tentar add silencioso.
