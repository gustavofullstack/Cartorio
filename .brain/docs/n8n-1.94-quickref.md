# N8N 1.94.x — Quick Reference

**Source**: https://docs.n8n.io/
**Versao em uso**: 1.94.x (flow.2notasudi.com.br)
**Atualizado**: 2026-06-25

## 1. Conceitos

- **Workflow (WF)**: grafo de nodes conectados. Ativado por trigger.
- **Node**: unidade de processamento (HTTP, Code, IF, Set, Telegram, etc).
- **Execution**: 1 rodada do WF. Status: success/error/running.
- **Credential**: secret armazenado (HTTP header, API key, OAuth).
- **Webhook**: URL publica que dispara WF ao receber POST.
- **Error Workflow**: WF acionado quando outro WF falha.

## 2. Endpoints API (N8N_API_KEY)

Base URL: `https://flow.2notasudi.com.br/api/v1`

```bash
N8N_KEY="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

# Listar todos WFs
curl -H "X-N8N-API-KEY: $N8N_KEY" https://flow.2notasudi.com.br/api/v1/workflows

# Listar execucoes
curl -H "X-N8N-API-KEY: $N8N_KEY" "https://flow.2notasudi.com.br/api/v1/executions?limit=10"

# Ativar/Desativar WF
curl -X POST -H "X-N8N-API-KEY: $N8N_KEY" \
  https://flow.2notasudi.com.br/api/v1/workflows/{id}/activate

# Deletar WF
curl -X DELETE -H "X-N8N-API-KEY: $N8N_KEY" \
  https://flow.2notasudi.com.br/api/v1/workflows/{id}
```

## 3. Nodes Essenciais

| Node | Uso | Auth |
|---|---|---|
| **HTTP Request** | chamar API externa | Header Auth |
| **Code** (JS/Python) | logica custom | n/a |
| **IF** | condicional | n/a |
| **Switch** | multi-branch | n/a |
| **Set** | criar/transformar variaveis | n/a |
| **Webhook** | receber POST externo | n/a |
| **Schedule Trigger** | cron | n/a |
| **Telegram** | enviar/receber msg Telegram | Bot Token |
| **Chatwoot** | enviar/receber msg Chatwoot | API Token |
| **Evolution API** (community) | WhatsApp | apikey |
| **Supabase** | query DB | service_role |
| **Postgres** | query Postgres | conn string |
| **Redis** | get/set cache | conn string |
| **Respond to Webhook** | retornar resposta HTTP | n/a |
| **Error Trigger** | capturar erro de outro WF | n/a |

## 4. Expressoes Comuns


## 5. Best Practices (cartorio)

- **Timeout 5s** em todos HTTP nodes (B08 - apply_n8n_timeouts.py)
- **Retry 3x exp backoff** (1s/5s/15s) em todos HTTP nodes (B07)
- **Error Workflow global** = WF 00-error-handler.json (B06)
- **Log estruturado JSON** com correlation_id (B09 - inject_n8n_correlation_logs.py)
- **Report Metrics N8N** no final de cada WF (B10 - inject_n8n_metrics_post.py)

## 6. Gotchas

- WFs exportados em JSON vao em `infra/n8n-workflows/`
- NAO criar WFs pela UI sem exportar para git (sync quebrado)
- Community nodes precisam de `n8n-nodes-evolution-api` instalado
- N8N Runner eh um pod separado que executa WFs (nao confundir com N8N principal)
- `n8n-nodes-base.code` precisa de `N8N_BLOCK_ENV_ACCESS_IN_NODE=false` para acessar env vars
