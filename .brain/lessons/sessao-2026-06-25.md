## 🏷️ Lessons por categoria

### Backend Hardening
- **L167**: WIP feature sem migration head pointer = porta aberta pra retrabalho
- **L168**: `backend/infra/` NÃO é path canonical. Infra sempre em `infra/` na raiz
- **L169**: Feature A26 adiciona notification fields ao Cliente. Validar LGPD art. 37
- **L170**: JWT HS256 requer secret >= 32 bytes (RFC 7518 sec 3.2) — validar no SERVICE, não no startup
- **L171**: Cursor pagination Relay: `end_cursor = None` quando `has_next_page=False` (sinaliza fim)
- **L172**: Pydantic v2 + Optional[str] = Field(default=None, min_length=32) valida mesmo quando None — usar sem min_length OU validar no service

### N8N / Workflows
- **L173**: N8N usa `maxTries` (v1.x) E `maxRetries` (legacy). Validator deve aceitar ambos
- **L174**: Webhook nodes não precisam de URL (path interno) — false positive em validators HTTP-only
- **L175**: Settings.errorWorkflow indica se WF tem error handler wired (B06 enforcement)

### TDD / Test Patterns
- **L176**: Tests que importam do conftest (db_session fixture) devem rodar DEPOIS conftest ser carregado
- **L177**: Cursor opaque base64 sem padding (`rstrip("=")`). Padding só pra decode

### Multi-Agent Coordination
- **L178**: Hooks auto-append (commits CI) podem criar commits intermediários fragmentando histórico do agent
- **L179**: Files duplicados em paths diferentes (git mv + filesystem copy) — sempre verificar md5 antes de commit
- **L180**: Working tree "ahead 1" pode ser falso se CI já fez push — sempre `git status -sb` antes de push

### Workflow Otimização
- **L181**: Trabalhador sequencial 1-agent render MAIS que paralelo quando há pytest fixtures compartilhadas
- **L182**: Validator descobrindo bugs REAIS em 41+ WFs (B07/B08) tem valor maior que código novo

## ⚠️ Anti-patterns identificados

### AP1: Deletar working tree "lixo" sem análise prévia
- **Exemplo**: 88 arquivos `.bak/.bak2` que PARECIAM lixo eram output de scripts one-shot
- **Lição**: Verificar md5 + git log antes de deletar — `.bak` pode ser backup de migração (ex: `cfb0cbc`)

### AP2: Migration sem `down_revision` definido
- **Exemplo**: A26 migration com `down_revision = '<previous_revision>'` literal
- **Lição**: SEMPRE validar alembic head antes de criar nova migration

### AP3: Workflow JSON duplicado em paths diferentes
- **Exemplo**: A26 workflows em `backend/infra/` (errado) + `infra/n8n-workflows/` (canônico)
- **Lição**: Path canonical de N8N workflows é SEMPRE `infra/n8n-workflows/`

### AP4: N8N WF sem error handler wired
- **41/45 WFs** detectados pelo B12 violam isso — `settings.errorWorkflow` não setado
- **Lição**: Toda WF nova deve ter error handler explicitamente wired

### AP5: Webhook nodes com URL hardcoded
- Vários WFs usam `https://api.example.com` ao invés de `{{$env.CARTORIO_API_URL}}`
- **Lição**: Parametrizar SEMPRE via `$env.VAR`

### AP6: Commits fragmentados por hooks CI
- CI agent `Cartorio CI <ci@cartorio.dev>` faz commits intermediários
- **Lição**: Aceitar como feature, não tentar reescrever — agent WORK é o que importa
