## Lições Aprendidas (L244+)

### L244: Retry com backoff 2s é SWEET SPOT para LLM blip (TURN 47)
- LLM pequeno pode ter blip (timeout gateway, parse truncado, connection reset)
- 1 em 20 chamadas tem blip, demais funcionam em 5-15s
- Retry 2x com backoff 2s cobre blip transitório sem degradar UX significativamente
- Mais que 2 tentativas = degrada UX (espera 4s+); user prefere fallback message
- **REGRA**: chat webhooks com LLM SEMPRE devem ter retry 2x com backoff
